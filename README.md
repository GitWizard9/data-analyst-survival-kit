# 🎒 The Data Analyst's Survival Kit

Willkommen beim großen Git-Abschlussprojekt! 🎉 

Dieses Repository ist unser gemeinsames "Überlebenshandbuch" für den IT-Alltag. Wenn wir fertig sind, enthält es geniale Python-Tricks, Lebensretter-SQL-Queries, tiefgründige Tech-Haikus und Bilder von Haustieren, die uns beim Debuggen helfen. 

Am Ende dient dir dieses Projekt als **echtes Open-Source-Portfolio** für dein GitHub-Profil. Du beweist damit, dass du den Git-Workflow im Team beherrschst.

---

## 🎯 Deine Mission

Bitte lies dir diese Anleitung **Schritt für Schritt** durch, bevor du startest. Du arbeitest hier genau so, wie Entwickler in echten Open-Source-Projekten arbeiten.

### Schritt 1: Issue sichern
1. Gehe oben auf den Reiter **Issues**.
2. Such dir eine Aufgabe aus, die dir gefällt.
3. Klicke im Issue auf **"Assign yourself"**, damit die anderen wissen: *Das ist meins!*

### Schritt 2: Forken & Klonen
1. Erstell einen Fork dieses Repositories in deinem eigenen GitHub-Account.
2. Klone **deinen Fork** auf deinen Rechner.
3. Erstelle einen neuen Branch für deine Arbeit.

### Schritt 3: Die Aufgabe lösen
Erledige die Aufgabe aus deinem Issue. 
* Wenn du Code schreibst, lege die Datei im Ordner `/snippets` ab.
* Wenn du ein Bild hochlädst, speichere es im Ordner `/assets`.

### Schritt 4: Die "Hall of Fame"
Damit dein Beitrag gültig ist, musst du dich in unsere `CONTRIBUTORS.md` eintragen. 
**WICHTIGE REGEL:** Trage deinen Namen und dein GitHub-Profil **zwingend in Zeile 3** ein (direkt unter der Überschrift)! 

### Schritt 5: Commit & Push
Speichere deine Änderungen und pushe sie auf *deinen* Fork bei GitHub.

### Schritt 6: Der Pull Request (PR)
Gehe auf GitHub in *deinen* Fork. Dort siehst du jetzt einen grünen Button **"Compare & pull request"**. 
Klicke darauf, um deine Änderungen in dieses Original-Repository vorzuschlagen. Verlinke in der PR-Beschreibung dein Issue (z. B. "Closes #5").

---

## ⚔️ Der Endgegner: Der Merge-Konflikt

Da **jeder** von euch seinen Namen in Zeile 3 der `CONTRIBUTORS.md` einträgt, wird es ab dem zweiten Pull Request unweigerlich krachen. GitHub wird dir sagen: **"This branch has conflicts that must be resolved."**

Das ist kein Fehler – das ist Absicht! Willkommen in der Realität der IT. 😉

**Deine finale Aufgabe:**
Löse diesen Konflikt! Du darfst deinen Namen nicht löschen, aber den der anderen auch nicht. 
* Du kannst den Konflikt direkt im Web-Editor von GitHub lösen (Button: "Resolve conflicts").
* **ODER (Pro-Weg):** Du fügst lokal dieses Original-Repo als `upstream` hinzu, machst einen `git pull upstream main`, löst den Konflikt in VS Code/PyCharm, machst einen neuen Commit und pushst erneut.

Sobald der Konflikt gelöst ist und das Häkchen bei deinem PR grün leuchtet, wird dein Code in das Survival Kit aufgenommen! 🚀 Viel Erfolg!
